The Living Planet — guide

An offline, single-file atlas of ocean currents, winds and climate patterns — with live data loading when you're online.

Getting around

  • Drag to rotate the globe · scroll to zoom · click a spot for live weather, air quality and any loaded data values at that point.
  • Hold Alt and hover to read exact grid values of the topmost data overlay under the cursor.
  • The month slider (left panel) drives every seasonal layer — ITCZ, sea ice, monsoons, cyclone basins, pressure systems.
  • Esc closes any dialog.

Flow layers

  • Ocean currents — warm/cold currents, the Antarctic Circumpolar Current and the subtropical gyres, animated from real long-term-mean CMEMS velocity data.
  • Winds — trade winds, westerlies and polar easterlies from ERA5 reanalysis; jet streams ride their own 250 hPa field.
  • Each class has its own colormap picker, and the Settings section has sliders for particle count, speed, trail fade, width and opacity.
  • The WebGPU renderer toggle (bottom of the left panel) moves particle physics onto the GPU — smoother with high particle counts on capable machines, and it falls back to the standard renderer automatically.

Loading your own data

  • Drop a file — drag any NetCDF (.nc) or GRIB2 file anywhere onto the globe. Multi-timestep files become animatable layers.
  • Large NetCDF-4/HDF5 files — the in-browser HDF5 engine runs in a WebAssembly heap capped at 2GB, and the whole file has to fit inside that cap alongside its own decompression working memory, so files over ~1.7GB won't load. Classic NetCDF-3 and GRIB2 don't share this ceiling — only NetCDF-4/HDF5 does. If you hit it, split the file first, e.g. with NCO: ncks -d valid_time,0,199 file.nc part1.nc, then drop the parts in as a series (the app merges them by time automatically).
  • NOMADS loader (🌐 button under the title) — pick a model (GFS, GDAS, GEFS), date range, forecast hours and exact variables/levels; only those fields' bytes are fetched from NOAA and loaded as one animatable overlay. Hover any variable or level for a plain-English description.
  • Load from URL (left panel) — direct links to .nc/.grib2 files, ERDDAP endpoints, or NOMADS filter URLs.
  • A loaded u/v wind or current pair can drive the particle animation itself — see "Loaded Fields" in the layer list, with side-by-side compare against the embedded climatology.

Working with overlays

  • Each overlay card has: variable/level/time pickers, ▶ animation with adjustable fps, contours with adjustable count/style/step, colormap picker, and opacity/brightness/contrast/saturation sliders.
  • ☁ Preload frames — decodes every timestep into memory up front so animation and the timeline scrub without per-frame decode stutter (it tells you if the series would exceed the cache budget).
  • 📈 Trend — fits a per-cell linear trend over the series (full, splits, custom ranges, or rolling window). Slopes are normalized to real time (per year) when the file carries dates, with the detected cadence stated up front; optionally mask cells that aren't statistically significant.
  • ➗ Derived layer — combine overlays with a formula (A−B, wind speed √(A²+B²), ratios, potential temperature…). Each term binds to an overlay·variable·level·time; "Compare A · B" gives quick Pearson r / bias / RMSE before you commit. Derived layers are first-class: animate them, extract regions, even fit trends on them.
  • Timeline (top of Earth Systems) — the master clock: with no data loaded, its month chips drive the seasonal layers; once overlays are loaded it switches to real dates and keeps every layer's frame in sync while playing.

Teleconnection & statistical tools

  • Reached from the target/radar-style icon on any loaded overlay's tools row (tooltip: "Teleconnection tools") — it opens a menu of the four tools below. All of them need enough loaded timesteps (3–8+ depending on the tool) to run at all, and a lag sweep (Correlation, Multiple regression) additionally needs real calendar dates on every layer involved — without them, lag is disabled and the tool falls back to a single frame-position-aligned pass.
  • 🔗 Correlation map — click a reference point on any loaded field, then correlate that point's time series against every cell of a target field (the same field, for a self-teleconnection like ENSO-SST, or a different one, e.g. an SST index against precipitation). If both layers carry dates you can sweep a range of lags (e.g. −12 to +12 steps): positive lag tests whether the reference LEADS the target by that many steps, and you get one correlation map per step to scrub through like animation frames. Reading it: r runs −1 to +1 — the further from 0, the stronger the linear relationship at that lag. Correlation is not causation: a strong r only says the two series move together linearly, not why. A large daily record swept across many lags is a lot of arithmetic, so a frame stride option (use every Nth frame) is pre-set to keep the first look responsive — set it to 1 for the full-resolution result.
  • 🧮 Composite analysis — pick an index region (a preset like ENSO/IOD/NAO, a country, or a box you draw), and the tool splits the target field's matched timesteps into "index running high" vs. "index running low" (above/below its own mean by a threshold, in standard deviations of that index), then averages each group's anomaly. Produces three frames — the high composite, the low composite, and their difference — e.g. mean rainfall anomaly across El Niño years vs. La Niña years. It composites anomalies (deviation from the whole compositing period's own mean), not raw values, so a big constant background pattern (like the pole-to-equator SST gradient) doesn't swamp the smaller signal you're isolating.
  • 📊 EOF / PCA — finds the dominant spatial patterns of variability in a field over a region, with no reference index needed at all. Produces a handful of ranked "mode" maps (EOF1, EOF2…), each labeled with the % of that region's variance it explains, plus a matching principal-component (PC) time series per mode — reachable via Region extract while that mode's frame is selected. Run it on tropical Pacific SST and EOF1 typically comes back looking like ENSO on its own — that's the method discovering the pattern, not something told to look for it. Two things worth knowing before reading too much into a result: a mode's sign is arbitrary (a "warm east / cool west" pattern is mathematically identical to the same pattern flipped — check the paired PC time series if the sign matters to you), and beyond the first mode or two, EOFs can blend multiple real physical patterns together rather than isolate one.
  • 📈 Multiple regression — fits target = β₀ + β₁·pred₁ + β₂·pred₂ + … at every grid cell against up to 6 region-mean index predictors, so each predictor's β map shows its relationship with the target controlling for the other predictors — e.g. ENSO's effect on rainfall once IOD is already accounted for, which a plain correlation map can't separate out. Produces one frame per predictor for β (in the predictor's native units, or "per 1σ" with standardization on), one shared R² (fraction of variance all predictors jointly explain, 0–1), and one t-statistic and p-value frame per predictor; optionally mask cells where p ≥ 0.05 as insignificant. Like Correlation, a lag range can be swept so predictors lead the target by varying amounts, grouped per lag on the time slider. Predictors that are themselves highly correlated with each other (collinear) make β numerically unstable — the tool refuses to run rather than hand you a misleading number in that case.
  • Picking a region — the preset dropdown/list and the free-text "Country / region…" search both resolve a country name to the same border-accurate coastline (not just its bounding rectangle) wherever the app's atlas has that country; anything without a matching country becomes a plain lon/lat box. Selecting a region briefly flashes its outline on the globe so you can confirm what you picked before running.

Plots, regions & export

  • 📊 2D plots — every overlay opens plot windows: point time series (click a spot), Hovmöller diagrams (time–latitude or time–longitude), and region-mean series. Plot windows are draggable/resizable and follow the data as frames change.
  • Region extraction — draw a box on the globe for area-mean statistics and time series over the box; export any of it as CSV (single frame or all timesteps).
  • 📷 Export PNG captures the current view. 📏 Measure gives great-circle distance between two clicked points. Annotations pin dated notes to locations, tied to the timeline.

Custom fields & boundaries

  • Loaded Fields (layer list) — load a u/v velocity JSON to drive the particle animation itself: swap it in place of the embedded currents/winds, or ⇄ Compare to run both side-by-side on split particle pools with separate colormaps.
  • Loaded boundaries (Geography group) — drop custom boundary GeoJSON (EEZs, basins, study regions) to draw it on the globe and toggle it like any other layer.

Appearance & performance

  • The Appearance tab holds themes and per-class colormaps for currents/winds/jets, particle sliders (count, speed, trail fade, width, opacity), label size/brightness boost, and night shading.
  • WebGL2 / WebGPU toggles (bottom of the panel): WebGL2 renders the data overlays; WebGPU moves the particle physics itself onto the GPU — smoother at high particle counts, with automatic fallback if unavailable.
  • If animation stutters with several overlays: preload frames, or reduce particle count / disable contours while playing.

Live feeds & tools

  • Live data feeds — USGS earthquakes (filter by magnitude/period) and coastal marine conditions, plus the point-click weather lookup (Open-Meteo).
  • Click any plotted feed marker for details and charts; click anywhere else for weather, air quality and loaded-data values at that point.
Tip: everything except the live feeds and the NOMADS loader works fully offline — the atlas, flow data and all analysis tools are embedded in this single file.
